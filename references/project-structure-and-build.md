# Project Structure and Build System

Complete reference for Connect IQ project configuration, manifest, jungle files, resources, and the build/deployment process.

## Project Directory Structure

```
my-project/
  manifest.xml              ' Application metadata, permissions, target devices
  monkey.jungle             ' Build configuration (entry point for the compiler)
  source/
    MyApp.mc                ' AppBase subclass — application entry point
    MyView.mc               ' Main view (extends View, WatchFace, DataField, etc.)
    MyDelegate.mc           ' Input delegate (extends BehaviorDelegate)
    MyGlanceView.mc         ' Glance view (CIQ 3.1.3+)
    MyMenuDelegate.mc       ' Menu input handler
    MyBackgroundService.mc  ' Background service delegate
  resources/
    strings/
      strings.xml           ' Default language strings
    drawables/
      drawables.xml         ' Bitmap and shape resources
      icon.png              ' Launcher icon (typically 40x40 or 60x60)
    layouts/
      layout.xml            ' UI layouts
    menus/
      menu.xml              ' Menu definitions (Menu2)
    properties.xml          ' App properties and settings definitions
    settings/
      settings.xml          ' Settings UI configuration (shown in Garmin Connect)
  resources-round/          ' Round screen overrides
  resources-rectangle/      ' Rectangle screen overrides
  resources-eng/            ' English language resources
  resources-deu/            ' German language resources
  barrel/                   ' External barrel (library) dependencies
```

### App Type Values

| Type | manifest value | Description |
|---|---|---|
| Watch Face | `watchface` | Custom watch face |
| Data Field | `datafield` | Activity data field |
| Widget | `widget` | Widget in widget loop |
| Device App | `watch-app` | Full application |
| Audio Content Provider | `audio-content-provider-app` | Audio playback |

### Application Attributes

| Attribute | Required | Description |
|---|---|---|
| `entry` | Yes | Class name of AppBase subclass |
| `id` | Yes | Unique UUID (generated by IDE) |
| `launcherIcon` | Yes | Resource reference for launcher icon |
| `minSdkVersion` | Yes | Minimum Connect IQ SDK version |
| `name` | Yes | Resource reference for app name |
| `type` | Yes | App type (see table above) |

### Common Permission IDs

| Permission | Enables |
|---|---|
| `Sensor` | Heart rate, temperature, accelerometer |
| `Communications` | HTTP requests, phone messaging |
| `Positioning` | GPS location |
| `Background` | Background service execution |
| `BluetoothLowEnergy` | BLE peripheral communication |
| `Ant` | ANT/ANT+ wireless |
| `UserProfile` | User profile data access |
| `FitRecording` | Activity recording to FIT files |
| `SensorHistory` | Historical sensor data |
| `Weather` | Weather conditions |
| `Complications` | Provide data to watch faces |
| `PushNotification` | Receive push notifications |

## monkey.jungle

The jungle file is the build system configuration. It defines source paths, resource directories, and per-device overrides.

### Basic Jungle File

```jungle
# Default project configuration
project.manifest = manifest.xml

# Source files
base.sourcePath = source

# Resources
base.resourcePath = resources

# Device-specific resource overrides
round.resourcePath = $(round.resourcePath);resources-round
rectangle.resourcePath = $(rectangle.resourcePath);resources-rectangle

# Exclude files for specific builds
base.excludeAnnotations = test

# Barrel (library) path
base.barrelPath = barrel
```

### Device-Specific Overrides

```jungle
# Override source for specific device
fenix7.sourcePath = source;source-fenix7
venu2.sourcePath = source;source-amoled

# Override resources for specific device
fenix7.resourcePath = resources;resources-fenix7

# Exclude annotations per device
round.excludeAnnotations = rectangular_only
```

### Build Annotations

```jungle
# Exclude annotations from release builds
base.excludeAnnotations = test;debug

# Include only for debug builds
debug.excludeAnnotations = release
```

## Barrels (Libraries)

Barrels are reusable Monkey C libraries packaged as `.barrel` files.

### Using a Barrel

1. Add to manifest.xml:
```xml
<iq:barrels>
    <iq:depends name="MyBarrel" version="1.0.0" />
</iq:barrels>
```

2. Add barrel path to monkey.jungle:
```jungle
base.barrelPath = barrel
```

3. Place `.barrel` file in `barrel/` directory

4. Use in code:
```mc
using MyBarrel.MyModule;
```

### Creating a Barrel

1. Create a new "Barrel" project type in VS Code
2. Define public API via modules and classes
3. Export as `.barrel` file using "Export Project" command

## Conditional Compilation

### Source Annotations

```mc
' Only included in debug builds
(:debug)
function debugLog(msg as String) as Void {
    System.println(msg);
}

' Only included in release builds
(:release)
function getEndpoint() as String {
    return "https://api.production.com";
}

' Only included when running tests
(:test)
function testAdd(logger as Logger) as Boolean {
    return add(2, 3) == 5;
}

' Background-safe code
(:background)
class MyService extends System.ServiceDelegate { }

' Exclude for specific device
(:exclForDevice("vivoactive3"))
function notForVivoactive3() as Void { }
```

### Jungle Exclusions

```jungle
' Exclude test annotations in release
release.excludeAnnotations = test;debug

' Exclude device-specific annotations
fenix5.excludeAnnotations = no_fenix5
```

### Runtime Feature Detection

```mc
' Check API availability before use
if (Toybox.WatchUi has :GlanceView) {
    ' GlanceView available (CIQ 3.1.3+)
}

if (Toybox.Background has :registerForTemporalEvent) {
    ' Background services available
}

if (Toybox.Weather has :getCurrentConditions) {
    ' Weather API available
}

if (Toybox.Graphics.Dc has :setAntiAlias) {
    ' Anti-aliasing available (AMOLED)
}
```

## Development Workflow

### VS Code Setup

1. Install the **Monkey C** extension from VS Code Marketplace
2. Install the Connect IQ SDK Manager
3. Download the SDK and device simulators via SDK Manager
4. Run `Monkey C: Verify Installation` command in VS Code

### Build and Run

```
VS Code Command Palette:
  Monkey C: Build Current Project      ' Compile
  Monkey C: Run                        ' Build and launch simulator
  Monkey C: Run Tests                  ' Run unit tests
  Monkey C: Export Project             ' Create .iq file for submission
  Monkey C: Edit Supported Products    ' Configure target devices
```

### Command Line Build

```bash
' Compile for simulator
monkeyc -d <device_id> -f monkey.jungle -o output.prg -y developer_key.der

' Compile for device (release)
monkeyc -d <device_id> -f monkey.jungle -o output.prg -y developer_key.der -r -e

' Run tests
monkeyc -d <device_id> -f monkey.jungle -o output.prg -y developer_key.der -t

' Export .iq package
monkeyc -e -f monkey.jungle -o output.iq -y developer_key.der -r
```

### Developer Key

A `.der` key file is required to sign apps. Generated on first build or via:
```
Monkey C: Generate Developer Key
```

## App Submission

### Pre-Submission Checklist

1. Update `manifest.xml` with all target devices
2. Ensure `minSdkVersion` is set correctly
3. Add all supported languages
4. Create a proper launcher icon (follow Garmin brand guidelines)
5. Test on the simulator for each target device
6. Test on at least one physical device

### Export and Submit

1. Run `Monkey C: Export Project` in VS Code
2. Select languages to include
3. A `.iq` file is generated containing all device binaries
4. Upload to https://apps.garmin.com/en-US/developer/upload
5. Fill in app description, screenshots, categories
6. Submit for review

### App Review Guidelines Key Points

- Must not crash or freeze
- Must handle sensor data unavailability gracefully (null checks)
- Must respect device limitations (memory, battery)
- Must not drain battery excessively
- Must provide meaningful user experience
- Must follow Garmin branding guidelines
- Description must accurately reflect functionality
- Screenshots must be representative
